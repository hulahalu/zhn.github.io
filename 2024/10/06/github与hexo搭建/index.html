github与hexo搭建

关于这个搭建过程，我首先在csdn上搜索教程。在上面教我创建了这个GitHub账号这个是英文网站，还时不时打不开 ，然后去网上搜索，教我更改什么IP地址，去以管理员身份更改hosts什么的来尝试，但并没有好转，最后通过VPN打开。然后是安装git，还有NodeJS。接下来使用Git Bash初始化环境，通过上面教程在电脑硬盘上存放了个文件来装hexo后面所需要的内容，进入文件夹右键，在右键菜单中选择Open Git Bash here，打开Git Bash窗口，照着教程上面的指令，一个一个的把它输进去，进行对hexo的安装，接下来我启动hexo的服务，要去一个网站http://localhost:4000/当我输入这个网址的时候会出现无法连接的状况，我就去网上搜索该如何进行访问，然后我得知可以改变路径的方法，我照上面的操作可并没有解决问题，我接着去git bash输入hexo server提示我有一个文件无法访问，，我就去上网搜索如何打开里面权限，通过打开了开发者模式就可以了，也是意外之喜。 然后我要创建一个git hub仓库，你照上面的要求进行勾选，还要为这个账号设置ssh继续在git bash输入上面的代码，可是我在实际操作中，这上面会无法读取我的路径，无法存储这个key他上面有出现乱码是在我的用户名称上，可能与我用户名称为中文有关系，我就更改了用户名称，可是系统原本中文名字文件并没有更改，不知道为什么，当我再次尝试时又能读取那个中文接着我就按着上面的那个密钥文件复制到git hub setting，我在git bash上检测公钥设置是否成功。 接着要我将博客部署到git hub在_config.yml文件里更改deploy，教程上面有个要输入用户名/用户名的代码我以为是同样的就输进去，结果一直报错，后面看到结尾带github.io.git那个就是我仓库的名称，更改后就可以在git bash输入hexo g和hexo d就成功完成了。
